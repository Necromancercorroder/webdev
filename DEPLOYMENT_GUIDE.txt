OPTION A: VERCEL (Recommended - Easiest)
-----------------------------------------
Frontend + Backend in one deployment

1. Install Vercel CLI:
   npm install -g vercel

2. Login to Vercel:
   vercel login

3. Deploy:
   vercel

4. Follow prompts and deploy!

Your app will be live at: https://your-project.vercel.app


OPTION B: NETLIFY (Frontend) + RENDER (Backend)
------------------------------------------------
FRONTEND (Netlify):
1. Go to https://netlify.com
2. Click "Add new site" > "Import an existing project"
3. Connect your Git repository
4. Build settings:
   - Build command: npm run build
   - Publish directory: dist
5. Add environment variable:
   - VITE_API_URL=https://your-backend-url.onrender.com
6. Deploy!

BACKEND (Render):
1. Go to https://render.com
2. Click "New +" > "Web Service"
3. Connect your repository
4. Settings:
   - Build Command: npm install
   - Start Command: npm start
   - Environment: Node
5. Add environment variables:
   - PORT=5000
   - JWT_SECRET=your_secret_key_here
   - NODE_ENV=production
6. Deploy!

After both deployments:
- Copy your Render backend URL
- Update VITE_API_URL in Netlify environment variables
- Redeploy Netlify


OPTION C: RAILWAY (Full Stack)
-------------------------------
1. Go to https://railway.app
2. Click "New Project" > "Deploy from GitHub repo"
3. Select your repository
4. Add environment variables:
   Backend:
   - PORT=5000
   - JWT_SECRET=your_secret_key_here
   - NODE_ENV=production
   
   Frontend:
   - VITE_API_URL=${{RAILWAY_PUBLIC_DOMAIN}}/api
5. Deploy!


OPTION D: FLY.IO (Full Stack)
------------------------------
1. Install Fly CLI:
   PowerShell: iwr https://fly.io/install.ps1 -useb | iex

2. Login:
   fly auth login

3. Deploy backend:
   cd backend
   fly launch
   
4. Deploy frontend:
   cd ..
   fly launch
